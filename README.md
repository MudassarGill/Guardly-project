# Guardly: Parental Monitoring Application

Guardly is a Flutter and FastAPI based application designed to help parents monitor, secure, and manage their children's device usage. This project is currently built up to the fundamental stages of User Authentication and Device Pairing.

## 🚀 Features Implemented So Far

1. **Modern Authentication UI**: 
   - Interactive Onboarding Screen.
   - Smooth Login and Sign-up forms linked to the backend API.
   - Custom loading indicators and visual success/error Toasts.

2. **Secure FastAPI Backend**:
   - SQLite Database setup (`models.py`, `database.py`).
   - Secure hashed passwords using `bcrypt` and `passlib`.
   - JWT (JSON Web Token) based endpoints for login and registration.

3. **Parent-Child Account Pairing**:
   - **Account Type Selection**: Users can choose to be a "Parent" or a "Child" during the setup process.
   - **Dynamic OTP Generation**: The Parent profile communicates with the backend to generate a fresh, secure 6-digit OTP code to share with the child.
   - **Child Device Linking**: The Child profile inputs the 6-digit code. The backend verifies the correct code and permanently links the Child account to the Parent account in the database (`ParentChild` table).

## 🛠 Tech Stack

- **Frontend**: Flutter (Dart), GetX (Routing & State Management), HTTP (API networking).
- **Backend**: Python, FastAPI, SQLAlchemy, SQLite, Pydantic.

## 📱 How the App Works (The Flow)

1. **Start**: The user opens the app and sees the animated Onboarding screen.
2. **Registration**: The user signs up by providing their First Name, Last Name, Email, and Password. These are stored locally via `uvicorn` backend.
3. **Login**: After registering, the user signs in. The app saves their `user_id`.
4. **Role Selection**: The user selects whether they are a Parent or a Child.
   - **If Parent**: Generating a code dynamically. They land on the "Share QR/OTP" screen. The app contacts the backend to generate a random 6-digit number, which is displayed on the screen.
   - **If Child**: Capturing the code. They land on the Scanner / Referral Screen. They tap the fields, enter the 6 digits generated by the Parent, and hit Continue.
5. **Success**: Once the code is submitted, the backend verifies it and links both accounts in the system!

## ⚙️ How to Run Locally

### Start the Backend
1. Open up your terminal in the backend directory.
2. Ensure you have installed requirements: `pip install -r requirements.txt`.
3. Start the FastAPI server: 
   ```bash
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```

### Run the App
1. The app depends on your local IP address for the Android device to talk to the Backend server. Ensure the IP address inside `lib/backend/api_service.dart` (currently configured as `192.168.1.9`) is your active Wi-Fi IPv4 address.
2. Run your Flutter app onto a device:
   ```bash
   flutter run
   # OR build an APK:
   flutter build apk --release
   ```
